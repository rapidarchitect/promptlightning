Powerful Features

Type-Safe Templates

Define inputs with strict typing: string, number, boolean, array, and object types with validation and coercion.

File-Based Management

Organize templates in YAML files with clear structure, making them easy to version control and collaborate on.

Hot Reload

Automatic template reloading during development with built-in file watching for rapid iteration.

Jinja2 Templating

Full Jinja2 support with custom filters like 'yaml' and 'default' for powerful template composition.

Semantic Versioning

Built-in version management with semantic versioning support and easy version bumping via CLI.

Execution Logging

Optional SQLite logging of template executions for debugging and analytics with zero configuration.

CLI Interface

Complete command-line interface for template management, listing, version control, and development workflow.

Thread-Safe Caching

Production-ready with thread-safe template caching using RLock for high-performance concurrent access.

Quick Start

Get up and running with Dakora in minutes

1Initialize Project

dakora init

Creates configuration file and example templates

2Create Template

id: greeting
version: 1.0.0
description: A personalized greeting template
template: |
  Hello {{ name }}!
  {% if age %}You are {{ age }} years old.{% endif %}
  {{ message | default("Have a great day!") }}
inputs:
  name:
    type: string
    required: true
  age:
    type: number
    required: false
  message:
    type: string
    required: false

3Use in Python

from dakora import Vault

# Initialize vault
vault = Vault("dakora.yaml")

# Get and render template
template = vault.get("greeting")
result = template.render(name="Alice", age=25)
print(result)

Real-World Examples

FastAPI + OpenAI Integration

Perfect for building LLM-powered APIs with structured prompts and type validation. 

from fastapi import FastAPI
from dakora import Vault
from openai import OpenAI

app = FastAPI()
vault = Vault("dakora.yaml")
client = OpenAI()

@app.post("/chat")
async def chat_endpoint(message: str, template_id: str):
    template = vault.get(template_id)

    result = template.run(
        lambda prompt: client.chat.completions.create(
            model="gpt-4",
            messages=[{"role": "user", "content": prompt}]
        ).choices[0].message.content,
        message=message
    )

    return {"response": result}

Development Workflow

Hot-reload templates during development for rapid iteration and testing. 

from dakora import Vault
from dakora.watcher import Watcher

vault = Vault("dakora.yaml")
watcher = Watcher("./prompts", on_change=vault.invalidate_cache)
watcher.start()

# Templates reload automatically when files change
template = vault.get("my_template")
result = template.render(input_text="Hello world")

CLI Workflow

Manage templates, versions, and development workflow from the command line. 

# List all templates
dakora list

# Get template content
dakora get summarizer

# Bump version (patch, minor, major)
dakora bump summarizer --minor

# Watch for changes during development
dakora watch
